# 淨零排放生活轉型互動模擬：程式設計規劃書

## 1. 專案目標

建立一個單一 HTML 檔案的互動式網頁，讓大學生透過按鈕操作「人民購買商品」與「企業販售商品」的過程，觀察肉類、蔬菜、電器三種商品在買賣過程中造成的碳排量變化。

本階段只做最小可運作版本，也就是：

- 使用單一 HTML 檔案，內含 CSS 與 JavaScript。
- 介面使用繁體中文。
- 支援手機、平板、電腦瀏覽。
- 使用者可按 A、B、C 讓人民購買不同商品。
- 使用者可按 1、2、3 讓企業販售不同商品。
- 畫面即時顯示碳排量、交易紀錄與簡單狀態變化。

## 2. 使用者對象

主要使用者是大學生，應以課堂示範、工作坊體驗、分組討論為情境設計。

介面應避免太多專業術語，重點是讓學生快速理解：

- 不同商品類型有不同碳排係數。
- 消費行為與企業供給都會影響整體碳排。
- 生活轉型與企業轉型需要互相配合。

## 3. 核心概念

系統中有兩個角色：

- 人民：負責購買商品。
- 企業：負責販售商品。

商品分為三類：

- 肉類：碳排量最高。
- 蔬菜：碳排量最低。
- 電器：單次碳排量中高，並可表現生活用品與能源消耗的關聯。

每次人民購買或企業販售時，都會增加總碳排量。畫面需立即更新，讓學生看到不同選擇造成的差異。

## 4. 最小可運作功能

### 4.1 人民購買商品

畫面提供三個主要按鈕：

- A：人民購買肉類
- B：人民購買蔬菜
- C：人民購買電器

按下後應發生：

- 增加對應商品的人民購買次數。
- 增加總碳排量。
- 更新人民角色狀態文字。
- 新增一筆交易紀錄。

### 4.2 企業販售商品

畫面提供三個主要按鈕：

- 1：企業販售肉類
- 2：企業販售蔬菜
- 3：企業販售電器

按下後應發生：

- 增加對應商品的企業販售次數。
- 增加總碳排量。
- 更新企業角色狀態文字。
- 新增一筆交易紀錄。

### 4.3 碳排量顯示

畫面需顯示：

- 目前總碳排量。
- 人民購買造成的累積碳排量。
- 企業販售造成的累積碳排量。
- 三種商品各自累積的碳排量。

單位建議使用「kg CO2e」。

### 4.4 交易紀錄

每次操作後，在交易紀錄區新增一筆文字，例如：

- 人民購買肉類，增加 7.0 kg CO2e。
- 企業販售蔬菜，增加 1.0 kg CO2e。

交易紀錄只需保存最近 8 至 10 筆，避免手機畫面過長。

### 4.5 重置功能

提供「重新開始」按鈕。

按下後：

- 所有碳排數值歸零。
- 商品購買與販售次數歸零。
- 清空交易紀錄。
- 角色狀態回到初始文字。

## 5. 建議碳排係數

此版本使用簡化教學數值，不需追求真實精準，但應符合直覺差異。

| 商品 | 人民購買碳排 | 企業販售碳排 | 說明 |
| --- | ---: | ---: | --- |
| 肉類 | 7.0 kg CO2e | 6.0 kg CO2e | 代表畜牧、冷鏈、運輸等排放 |
| 蔬菜 | 1.2 kg CO2e | 1.0 kg CO2e | 代表低碳飲食選擇 |
| 電器 | 5.0 kg CO2e | 4.5 kg CO2e | 代表製造、物流、能源相關排放 |

實作時可將數值集中放在 JavaScript 物件中，方便後續調整。

## 7. 主要函式規劃

建議實作以下函式：

```javascript
function handleConsumerBuy(productId) {}
function handleBusinessSell(productId) {}
function addEmission(source, productId, amount) {}
function addLog(message) {}
function render() {}
function resetSimulation() {}
function formatEmission(value) {}
```

### 函式職責

- `handleConsumerBuy(productId)`：處理人民購買事件。
- `handleBusinessSell(productId)`：處理企業販售事件。
- `addEmission(source, productId, amount)`：統一更新碳排數值。
- `addLog(message)`：新增交易紀錄並限制筆數。
- `render()`：將 state 內容更新到畫面。
- `resetSimulation()`：重置所有狀態。
- `formatEmission(value)`：統一格式化數字，例如保留 1 位小數。

## 8. 介面版面規劃

### 8.1 整體區塊

畫面可分為五個區塊：

1. 頁首標題區
2. 碳排總覽區
3. 角色操作區
4. 商品碳排統計區
5. 交易紀錄區

### 8.2 頁首標題區

內容建議：

- 標題：淨零生活轉型互動模擬
- 副標：觀察人民消費與企業販售如何影響碳排量

### 8.3 碳排總覽區

顯示三個重點數字：

- 總碳排量
- 人民累積碳排
- 企業累積碳排

此區應放在畫面上方，讓學生每次操作後都能立刻看見變化。

### 8.4 角色操作區

左右或上下分成兩組：

- 人民角色
  - A 買肉類
  - B 買蔬菜
  - C 買電器
- 企業角色
  - 1 賣肉類
  - 2 賣蔬菜
  - 3 賣電器

在桌機版可使用兩欄排列；手機版改為單欄上下排列。

### 8.5 商品碳排統計區

以三個小卡或列表顯示：

- 肉類累積碳排
- 蔬菜累積碳排
- 電器累積碳排

每個商品可顯示：

- 累積碳排量
- 人民購買次數
- 企業販售次數

### 8.6 交易紀錄區

顯示最近操作紀錄。

若沒有紀錄，顯示：

- 尚無交易紀錄，請按下按鈕開始模擬。

## 9. RWD 設計規範

### 9.1 手機

- 單欄排版。
- 按鈕寬度接近滿版，方便觸控。
- 數字區塊垂直排列。
- 交易紀錄不應佔滿整個畫面，可設定最大高度與捲動。

### 9.2 平板

- 可使用兩欄排版。
- 人民與企業操作區並排或以等寬卡片呈現。
- 統計卡片可使用 2 至 3 欄。

### 9.3 電腦

- 使用置中的最大寬度容器，例如 `max-width: 1100px`。
- 總覽區可三欄排列。
- 人民與企業操作區兩欄排列。
- 商品統計區三欄排列。

## 10. 視覺風格建議

整體應清楚、乾淨、適合教學使用。

建議方向：

- 背景使用淺灰或接近白色。
- 主要色可用綠色，呼應淨零與永續。
- 警示或高碳排可用橘紅色輔助。
- 按鈕應有明確 hover / active 狀態。
- 數字應放大，讓投影時也容易閱讀。

避免：

- 過多動畫。
- 過度裝飾。
- 需要登入、資料庫或外部套件。

## 11. DOM 結構建議

單一 HTML 可使用以下大致結構：

```html
<main class="app">
  <header class="hero"></header>

  <section class="summary"></section>

  <section class="actions">
    <article class="role-panel consumer-panel"></article>
    <article class="role-panel business-panel"></article>
  </section>

  <section class="product-stats"></section>

  <section class="activity-log"></section>

```

建議使用 `data-product` 與 `data-action` 綁定按鈕事件：

```html
<button data-action="consumer-buy" data-product="meat">A 買肉類</button>
<button data-action="business-sell" data-product="meat">1 賣肉類</button>
```

## 12. 互動流程

### 12.1 人民購買流程

1. 使用者按下 A、B、C 其中一個按鈕。
2. JavaScript 讀取按鈕上的商品類型。
3. 系統取得該商品的人民購買碳排係數。
4. 更新人民累積碳排、商品累積碳排、總碳排。
5. 更新商品購買次數。
6. 新增交易紀錄。
7. 重新渲染畫面。

### 12.2 企業販售流程

1. 使用者按下 1、2、3 其中一個按鈕。
2. JavaScript 讀取按鈕上的商品類型。
3. 系統取得該商品的企業販售碳排係數。
4. 更新企業累積碳排、商品累積碳排、總碳排。
5. 更新商品販售次數。
6. 新增交易紀錄。
7. 重新渲染畫面。

## 13. 鍵盤操作加分項

最小版本可只做滑鼠與觸控操作。

若時間允許，可加入鍵盤事件：

- 按 A：人民買肉類
- 按 B：人民買蔬菜
- 按 C：人民買電器
- 按 1：企業賣肉類
- 按 2：企業賣蔬菜
- 按 3：企業賣電器

注意：按鍵提示應與按鈕文字一致。

## 14. 驗收標準

實作完成後，至少需符合以下條件：

- 開啟 HTML 後不需要伺服器即可使用。
- 手機寬度下沒有水平捲動。
- A、B、C 三個人民購買按鈕可正常增加碳排量。
- 1、2、3 三個企業販售按鈕可正常增加碳排量。
- 總碳排量等於人民累積碳排加企業累積碳排。
- 商品統計數字會隨操作更新。
- 交易紀錄會新增，且不會無限延伸。
- 重新開始可以清空所有狀態。
- 介面文字皆為繁體中文。

## 15. 後續可擴充方向

本階段不需要實作，但可保留擴充彈性：

- 加入低碳轉型按鈕，例如企業改用綠電、人民改買低碳商品。
- 加入預算、庫存或需求量。
- 加入碳排目標線，例如總碳排超過 50 kg CO2e 時顯示提醒。
- 加入圖表呈現碳排變化。
- 加入任務模式，例如「在 10 次交易內控制碳排低於 30 kg CO2e」。
- 加入角色回饋文字，引導學生思考消費與企業責任。
